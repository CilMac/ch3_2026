          Fonctionnalités

          Calcul

          Calcule les unités d'alcool d'une consommation à partir du volume (ou du poids) et du degré, avec des presets pour les boissons courantes (bière, vin, saké, porto, spiritueux…). Propose aussi les favoris en accès direct — un tap archive la conso récurrente immédiatement, ou tu peux la modifier avant d'archiver — et permet d'enregistrer la consommation dans l'historique (date, type de boisson, note), le tout sur un seul écran.

          Soirée

          Cumule les verres de la soirée en cours et estime indicativement l'alcoolémie (formule de Widmark simplifiée).

          Détail

          Consulte, trie et supprime les consommations déjà archivées.

          Synthèse

          Stats globales et séries sans consommation, bilan de la semaine sélectionnée, tendance sur les 12 dernières semaines, et bilan général par semaine — répartis en sous-écrans pour rester lisibles sur mobile.

          Configuration

          Jeton d'accès GitHub, export/import de sauvegarde, et réinitialisation complète de l'historique.

          Sous le capot

          CH3-CH2-OH — suivi personnel de consommation d'alcool. Page statique hébergée sur GitHub Pages ; les données vivent uniquement dans data/data.json de ce dépôt, synchronisées via l'API GitHub. Aucun tiers, aucun tracker.

          HTML/CSS/JavaScript natif, sans framework ni backend : toute la logique tourne dans le navigateur, la seule dépendance externe est l'API GitHub pour lire et écrire data/data.json.

          Calculs indicatifs : 1 unité = 10 g d'alcool pur (densité éthanol 0,8). L'alcoolémie estimée utilise une formule de Widmark simplifiée — elle ne remplace pas un éthylotest homologué.

          Configuration — synchronisation GitHub

          Colle ici ton jeton d'accès personnel GitHub (fine-grained, scope Contents: write limité au dépôt CilMac/ch3_2026). Il reste stocké uniquement sur cet appareil (localStorage), jamais committé ni envoyé ailleurs qu'à l'API GitHub.

          Export / Import

          Sauvegarde locale de secours, utile si le jeton pose souci ou pour migrer manuellement. La synchro normale se fait via GitHub — ceci n'est qu'un filet de sécurité.

          Réinitialisation

          Efface définitivement tout l'historique dans data/data.json. Action irréversible (pas de sauvegarde automatique) — pense à exporter une copie avant (ci-dessus).

          Test de synchronisation diagnostic

          Vérifie que la lecture et l'écriture de data/data.json fonctionnent bien dans les deux sens avant de passer à la suite.
